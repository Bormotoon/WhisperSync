**WhisperSync** синхронизирует звук и видео при **dual-system recording**: камера снимает видео с черновым звуком, а внешний рекордер (петличка, Zoom, Tascam, телефон с радиомикрофоном) пишет чистый звук отдельно. Два устройства работают от независимых кварцевых генераторов, поэтому за минуты и часы записи их время медленно расходится — ползучий, иногда нелинейный **дрейф часов** до секунды, который waveform-матчеры фиксируют в начале, но не умеют отслеживать дальше. WhisperSync находит точное выравнивание *по всей длине записи* и создаёт готовый к импорту **FCPXML**-проект для Final Cut Pro или DaVinci Resolve.

![WhisperSync GUI](docs/images/main_window.png)

> Главное окно: drag-and-drop источников, выбор стратегии, живой многодорожечный таймлайн и лог в реальном времени.

Пайплайн: **транскрипция** → **поиск словесных якорей** → **RANSAC-оценка K/offset** → **стратегия синхронизации** → **рендер синхронного аудио** → **экспорт FCPXML**.

Используя [faster-whisper](https://github.com/SYSTRAN/faster-whisper) (CTranslate2), WhisperSync транскрибирует оба аудиопотока с таймстемпами на уровне слов, находит совпадающие слова (якоря) через последовательностное выравнивание, применяет RANSAC-регрессию для устойчивой оценки коэффициента часов **K** и **смещения**, а затем одной из трёх стратегий пересобирает звук рекордера под картинку. Рендер сохраняет родные каналы и битность рекордера сквозь весь путь (никакого принудительного моно/16 бит) и использует прозрачный ресемплинг вместо time-stretch там, где реальный дрейф часов достаточно мал. Исходные медиафайлы никогда не изменяются.

## Возможности

### Движок синхронизации

- **Якоря на уровне слов** — оба трека транскрибируются с по-словными таймстемпами; общие слова становятся временными якорями. Работает там, где waveform-матчеры сдаются (эхо в камере, дальние микрофоны, шумные помещения).
- **Грубый-затем-точный матчинг для длинных источников** — клип сначала грубо локализуется в многочасовом рекордере голосованием по редким словам, затем точно матчится в узком окне.
- **RANSAC-регрессия + двухступенчатый фильтр выбросов** — устойчивая оценка K и offset, переживающая ошибки транскрипции и ложные совпадения; юникод-нормализация токенов сохраняет якоря на любом языке («ё», тире, кавычки — не помеха).
- **Три честные стратегии** — Global Linear, Local Time-Stretch и рекомендуемая Hybrid (фразы по якорям, паузы поглощают остаток). См. [гид](#гид-по-стратегиям).
- **Auto-strategy** — после каждого прогона характер дрейфа (residual, разброс локальных скоростей) сверяется с использованной стратегией; если лучше подошла бы другая — warning скажет, какая и почему.
- **Акустический fallback («Strategy 0»)** — клип *без единого* текстового совпадения ни с одним рекордером (музыка, шум, язык, который Whisper не берёт, почти-тишина) выравнивается грубым GCC-PHAT сканом по волновым формам, без слов.
- **Boundary Flex** — покадровое доуточнение губ-синка: старт каждого рендер-куска акустически поправляется кросс-корреляцией (GCC-PHAT) звука камеры и рекордера (вкл. по умолчанию).
- **Seam-snap-to-silence** — границы кусков в piecewise-стратегиях притягиваются к ближайшей паузе между словами рекордера, чтобы разрез никогда не попадал внутрь слова (никакого «заикания»).
- **Per-camera калибровка губ-синка** — постоянную задержку «микрофон↔губы», зашитую в аудиотракт конкретной камеры, не видит ни один акустический метод; `camera_av_offset_ms` (глобально или по камерам) её компенсирует.
- **Гейт по числу якорей** — клипу со слишком малым числом якорей не доверяется шаткий таймкод-фит; он укладывается по порядку имён с предупреждением, а не улетает в заведомо неверную позицию.

### Качество звука (bit-perfect рендер)

- **Родные каналы и битность сквозь весь путь** — никакого принудительного моно и 16 бит; стерео 24-бит на входе — стерео 24-бит на выходе. Куски режутся из лоссless PCM-мастера (заодно чинится несэмпло-точный seek в mp3/m4a-источниках).
- **Прозрачный ресемплинг вместо time-stretch** — реальный дрейф часов — доли процента; такие куски конформятся ресемплингом («varispeed», сдвиг питча на единицы центов — неслышимо на речи) вместо `atempo`/WSOLA с его фазовыми артефактами. WSOLA включается только когда куску действительно нужна большая коррекция.
- **Фейды только там, где нужны** — акустически непрерывные куски стыкуются встык; фейд (слышимая «ямка» громкости) ставится только на реально разрывные швы.
- **Сборка одним проходом** — конкатенация, точная длина и опциональный дакинг пауз в одном ffmpeg-энкоде, а не цепочке поколений; `libsoxr`-ресемплинг где доступен.
- **Дакинг пауз (опция)** — приглушает рекордер в паузах, где молчат *обе* дорожки (по полным спискам слов, не по зазорам якорей), скрывая рассинхрон эмбиента между фразами.

### Входы и выходы

- **Мультикамера** — клипы каждой камеры в своей подпапке; каждая камера на своей дорожке таймлайна, чистый звук синхронизируется один раз с камеры-референса (автовыбор или `--audio-source-camera`).
- **Несколько рекордеров** — `--audio-file` несколько раз; `--recorder-mode best` берёт лучший рекордер на клип (одна дорожка), `all` кладёт каждый рекордер на свою (мульти-микрофонные съёмки).
- **Экспорт FCPXML** (по умолчанию v1.9) — ссылки на нетронутые видеофайлы плюс рендеренные синхронные WAV, с честными атрибутами каналов/рейта каждого ассета; файл валидируется перед выдачей.
- **`--render-master-wav`** — дополнительно миксует все синхронные voice-клипы (и эмбиент, если включён) по их позициям в один WAV на весь таймлайн поверх тишины — для тех, кто работает без NLE.
- **Дорожка эмбиента (опция)** — AI-модель разделения источников убирает собственный (эхный, слегка рассинхронный) голос камеры, оставляя атмосферу помещения, на отдельной дорожке; работает в изолированном `.sep-venv`, батчем с одной загрузкой модели.
- **Экспорт транскриптов** — полные транскрипты каждого рекордера и клипа камеры сохраняются как JSON + SRT рядом с выходом (с таймстемпами слов).

### GUI

- **PyQt6-приложение с тёмной темой** и drag-and-drop зонами для папки видео, файла(ов) рекордера и папки вывода.
- **Мульти-рекордер** — бросьте несколько аудиофайлов сразу; при 2+ файлах разблокируется выбор режима `best`/`all`.
- **Живой многодорожечный таймлайн** — по строке на камеру и на аудиодорожку: реальная позиция каждого клипа, применённое изменение скорости (напр. `+0.10%`) и живой статус: pending (пунктир), working (оранжевая рамка), done (залит). Наведение — offset / duration / in-point / speed.
- **Диалог Transcription Settings** — модель, язык, устройство, compute type, режим транскрипции (fast/quality) и initial prompt без правки конфиг-файла.
- **Re-run with Selected Strategy** — после прогона переключите стратегию и запустите заново; транскрипты в кэше, пересинк идёт сразу в выравнивание/рендер.
- **Взвешенный общий прогресс** — одна непрерывная шкала на все стадии (без сбросов на каждой), плюс явный статус «Loading Whisper model…» при первой загрузке модели.
- **Предупреждения пайплайна в логе** — невыровненные клипы, высокий residual, совет по стратегии, проблемы валидации.
- **Отзывчивая отмена** — срабатывает посреди клипа, даже во время большого многоядерного рендера.
- **Встроенная вкладка Help** — учебник плюс интерактивный симулятор микро-синхронизации, показывающий, как каждая стратегия перестраивает звук при движении ползунков дрейфа и длины фразы.

### CLI и автоматизация

- **Полный headless-режим** — каждая настройка доступна флагом или JSON-конфигом; `--json` печатает машиночитаемый отчёт в stdout, прогресс — в stderr.
- **Осмысленные коды возврата** — `0` успех, `1` сбой выполнения, `2` ошибка аргументов/конфига.
- **`--dry-run`** — только скан + транскрипция + выравнивание, сводка без обработки звука.
- **`--verify`** — после рендера меряет *реализованный* губ-синк каждого клипа через GCC-PHAT и печатает median/p90/max (доступно и отдельно: `tools/verify_sync.py`).
- **Самопроверка окружения** — `python -m whispersync.engine.system_check`: ffmpeg, CUDA (тем же путём ctranslate2, что и движок), зависимости, место на диске; пишет `report.json`.

### Производительность и надёжность

- **Ускорение на NVIDIA GPU через ctranslate2** — torch не нужен; батчевый инференс — главный рычаг скорости, с автоматической OOM-лестницей (меньше батч → легче compute type → CPU).
- **Кэш транскрипций** — ключ SHA-256 по файлу, настройкам и *фактическому* (резолвнутому) device/compute type; повторные прогоны пропускают транскрипцию целиком. Опциональный прунинг по возрасту (`cache_max_age_days`).
- **Один общий рендер-пул** — куски всех клипов рендерятся на всех ядрах через единый пул процессов; сборка каждого клипа перекрывается с рендером следующих.
- **Boundary Flex в памяти** — оба трека декодируются в память один раз на клип, окна — слайсы массивов (никаких ffmpeg-спавнов на каждую границу).
- **Fork-безопасность** — рендер-пул форкается только в однопоточном процессе (CLI); GUI получает forkserver/spawn, что исключает классический дедлок форка многопоточного Qt-процесса.
- **Ранняя выгрузка VRAM** — модель Whisper выгружается сразу после выравнивания, освобождая GPU для рендера/сепарации.
- **Кроссплатформенность** — Windows, macOS, Linux; CI на Python 3.10–3.14 с lock-файлами для воспроизводимости.

## Как это работает

1. **Probe** — читаются длительность и аудиоформат (каналы, битность, sample rate) каждого клипа.
2. **Транскрипция** — Whisper превращает черновой звук камеры и звук рекордера в транскрипты с таймстемпами слов. Многочасовые рекордеры идут через батчевый GPU-инференс; результаты кэшируются.
3. **Поиск якорей** — общие слова обоих транскриптов становятся временными якорями. Сначала грубое голосование по редким словам локализует клип в рекордере, затем точный матчинг в узком окне.
4. **Фит** — RANSAC-регрессия оценивает коэффициент часов **K** и **offset**, отбрасывая ложные совпадения как выбросы (два этапа: грубое окно + пост-фит фильтр по резидуалам).
5. **Пересборка** — выбранная стратегия превращает выравнивание в рендер-«куски» (старт в рекордере, длительность, темп-фактор); seam-snap и Boundary Flex уточняют точки разреза; каждый кусок конформится прозрачным ресемплингом или atempo и собирается в один непрерывный WAV на клип, в родном качестве рекордера.
6. **Экспорт** — пишется FCPXML со ссылками на исходные видео и рендеренный синхронный звук, готовый для Final Cut Pro / DaVinci Resolve.

## Требования

| Компонент | Минимум | Рекомендуется |
|-----------|---------|---------------|
| Python | ≥ 3.10 | 3.12+ |
| GPU | нет (CPU fallback) | NVIDIA GPU (CUDA/cuDNN) |
| ffmpeg / ffprobe | в PATH | свежий билд с `libsoxr` |
| RAM | 4 GB | 8+ GB |
| Диск | 10 GB свободных | SSD |

- **NVIDIA GPU** рекомендуется для быстрой транскрипции. Транскрипция идёт через [faster-whisper](https://github.com/SYSTRAN/faster-whisper)/ctranslate2, которому **не нужен torch** — CUDA определяется собственной проверкой ctranslate2. Без GPU всё работает на CPU (медленнее).
- **ffmpeg/ffprobe** должны быть в `PATH` — извлечение аудио, ресемплинг/time-stretch и сборка. Билд с `libsoxr` даёт более качественный ресемплинг (при его отсутствии WhisperSync автоматически откатывается на встроенный).
- **CUDA/cuDNN** нужны только для GPU-режима — ставятся через [NVIDIA CUDA Toolkit](https://developer.nvidia.com/cuda-toolkit).

## Установка

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/Bormotoon/WhisperSync.git
cd WhisperSync

# 2. Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate   # Linux/macOS
# venv\Scripts\activate    # Windows

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Проверьте окружение
python -m whispersync.engine.system_check
```

`system_check` проверяет ffmpeg, CUDA (через ctranslate2 — тем же путём, что движок транскрипции в рантайме), Python, зависимости, опциональный `.sep-venv` (фича ambience-track) и свободное место. Печатает цветную таблицу и пишет `report.json` в текущий каталог.

**Опционально — дорожка эмбиента.** AI-сепарация живёт в изолированном окружении, чтобы её тяжёлые зависимости не трогали основную установку:

```bash
./setup_sep_venv.sh
```

## Быстрый старт

```bash
# GUI
python main.py

# CLI — всё по умолчанию (стратегия Hybrid, автоопределение устройства)
python main.py --cli --video-dir ./videos --audio-file recorder.wav
```

Бросьте папку с видео и файл(ы) рекордера в GUI, нажмите **SYNC** и импортируйте полученный `sync_output.fcpxml` в Final Cut Pro или DaVinci Resolve.

## Использование

### GUI

```bash
python main.py
```

- **Drag-and-drop** папки видео и аудиофайла(ов) рекордера — можно бросить несколько сразу; при 2+ файлах разблокируется выбор `best`/`all` (аналог `--recorder-mode`).
- Выбор стратегии (3 радиокнопки) и опций: timebase, crossfade, Boundary Flex, дакинг пауз, дорожка эмбиента.
- **Transcription Settings…** — диалог с моделью / языком / устройством / compute type / initial prompt / режимом транскрипции.
- **SYNC** запускает прогон; таймлайн, прогресс и лог обновляются вживую; **Cancel** срабатывает посреди клипа.
- После успешного прогона: **Open Output Folder** и **Re-run with Selected Strategy** (транскрипты в кэше — смена стратегии идёт сразу в выравнивание/рендер).
- При первой загрузке модели (возможно, со скачиванием весов с Hugging Face) строка статуса говорит об этом явно, а не выглядит зависшей.

### CLI

```bash
# Базовый запуск (стратегия по умолчанию — 3, Hybrid)
python main.py --cli --video-dir ./videos --audio-file rec.wav --output out.fcpxml

# Strategy 1 — Global Linear, JSON-отчёт для автоматизации
python main.py --cli --video-dir ./videos --audio-file rec.wav \
  --strategy 1 --json 2>/dev/null

# Мультикам (подпапки) + две петлички на отдельных дорожках
python main.py --cli --video-dir ./shoot \
  --audio-file lavA.wav --audio-file lavB.wav --recorder-mode all

# CPU, модель поменьше, только выравнивание
python main.py --cli --video-dir ./videos --audio-file rec.wav \
  --device cpu --compute-type int8 --model medium --dry-run

# Пост-рендер самопроверка губ-синка + единый мастер-WAV для работы без NLE
python main.py --cli --video-dir ./videos --audio-file rec.wav \
  --verify --render-master-wav
```

## Гид по стратегиям

| Стратегия | Название | Когда использовать |
|-----------|----------|--------------------|
| **1** | Global Linear | Линейный дрейф часов (самый частый случай). Один темп-конформ на весь файл. Быстрее всего, минимум обработки. |
| **2** | Local Time-Stretch | Нелинейный дрейф, «плавающий» темп. Каждый сегмент между якорями со своим коэффициентом; границы притянуты к паузам между словами. |
| **3** | Hybrid (Global + Silence) | Универсальная, **рекомендуемая по умолчанию**. Каждая фраза корректируется глобальным `K` клипа и ставится по своему якорю; пауза между фразами поглощает остаток. Устойчива к нелинейному дрейфу. |

```
Strategy 1        Видео:  |========================>
                  Аудио:  |========================>  × conform(1/K)

Strategy 2        Видео:  |=== сег1 ===|=== сег2 ===|=== сег3 ===>
                  Аудио:  |== сег1 ==>|==== сег2 ====|== сег3 ==>   (свой коэффициент на сегмент)

Strategy 3        Видео:  |== фраза ==| пауза |== фраза ==| пауза |== фраза ==>
                  Аудио:  |==×(1/K)===| тишина|==×(1/K)===| тишина|==×(1/K)==>
```

> **О «тональности».** Реальный дрейф часов — доли процента, и такие куски WhisperSync конформит прозрачным ресемплингом вместо time-stretch: питч сдвигается на ту же крошечную долю (единицы центов, неслышимо на речи) без фазовых артефактов WSOLA. `atempo` включается только когда фактическая коррекция куска превышает порог (`stretch_method` / `RESAMPLE_CONFORM_MAX_DEVIATION`).

> **Заикание на швах.** В piecewise-стратегиях (2, 3) границы кусков притягиваются к ближайшей паузе между словами рекордера (seam-snap-to-silence), а не режутся точно по таймкоду якоря — характерный мид-ворд stutter исчезает без потери точности синка.

> **Auto-strategy.** После каждого прогона измеренный характер дрейфа сверяется с использованной стратегией; если другая подошла бы лучше — warning скажет, какая и почему. Пересинк дешёвый — транскрипты в кэше.

> **Strategy 0 (акустический fallback).** Если у клипа нет ни одного пригодного текстового совпадения ни с одним рекордером (музыка, шум, неподдерживаемый язык), грубый GCC-PHAT скан по сетке оценивает offset/K напрямую по волновым формам — менее точно, чем словесные якоря, но полный отказ превращается в рабочее размещение, пока один и тот же физический звук доходит до обоих микрофонов.

> **Размещение клипов.** Каждый клип выравнивается к рекордеру независимо, позиция берётся из совпавших таймкодов — клипы не обязаны идти встык, реальные паузы между дублями сохраняются. Клип с числом якорей ниже `min_anchors` укладывается по порядку имён с предупреждением.

> **Мультикамера.** Клипы каждой камеры — в свою подпапку `--video-dir` (напр. `videos/camA/`, `videos/camB/`). Каждая камера на своей дорожке; чистый звук синхронизируется один раз с камеры-референса (`--audio-source-camera`, автовыбор по лучшему выравниванию), чтобы не задваиваться на углах. Видеофайлы в корне `--video-dir` при наличии подпапок-камер игнорируются с предупреждением.

> **Несколько рекордеров.** `--audio-file` несколько раз. Каждый клип выравнивается ко всем рекордерам; таймлайн строится по «основному» (лучшее покрытие). `--recorder-mode best` (по умолчанию) — одна дорожка, лучший рекордер на клип; `all` — каждый рекордер на своей дорожке (несколько петличек/спикеров). **Важно:** если файлы — просто последовательные куски *одного* устройства (рекордер бьёт запись по 15 мин), у них одни часы — склейте их заранее без потерь (`ffmpeg` concat), а не передавайте как разные рекордеры.

## Выходные файлы

Всё складывается рядом с FCPXML (по умолчанию — в папке видео):

| Путь | Что это |
|------|---------|
| `sync_output.fcpxml` | Файл проекта — импортируйте в Final Cut Pro / DaVinci Resolve |
| `audio_synced/<клип>_voice.wav` | Непрерывный синхронный voice-WAV на каждый клип камеры, в родном качестве рекордера |

## Проверка результата

- **`--verify`** — после успешного прогона меряет *реализованный* лаг между каждым рендеренным voice-WAV и звуком камеры через GCC-PHAT кросс-корреляцию, печатая median/p90/max по клипам (и включая цифры в `--json`-отчёт).
- **`tools/verify_sync.py`** — то же измерение как отдельный инструмент для любой пары аудиофайлов: `python tools/verify_sync.py camera.wav voice.wav [--json]`.
- **`python -m whispersync.engine.system_check`** — аудит окружения: ffmpeg, CUDA-через-ctranslate2, зависимости, `.sep-venv`, место на диске.

## Troubleshooting

### `CUDA not available, falling back to CPU`

Установите [NVIDIA CUDA Toolkit](https://developer.nvidia.com/cuda-toolkit) и cuDNN, затем запустите `python -m whispersync.engine.system_check` — он проверяет CUDA через ctranslate2, ровно тем путём, которым движок транскрипции пользуется в рантайме (torch не нужен).

### `ffmpeg not found in PATH`

- **Ubuntu/Debian:** `sudo apt install ffmpeg`
- **macOS:** `brew install ffmpeg`
- **Windows:** скачайте с [ffmpeg.org](https://ffmpeg.org/download.html) и добавьте в PATH

### Мало якорей

```
Warning: Only 3 anchors found (minimum: 8)
```

Причины и решения: очень короткий клип (больше речи — лучше), сильный шум (держите `vad_filter` включённым), несовпадающие языки (укажите `--language`), тихий сигнал рекордера (проверьте уровни). Клип ниже `min_anchors` укладывается по порядку имён с предупреждением, а не доверяется шаткому фиту. Если текстовых совпадений нет *вообще* — включается акустический fallback (`acoustic_fallback`, вкл. по умолчанию): волновой GCC-PHAT скан, грубее, но работает на музыке/шуме/тишине, пока один и тот же физический звук доходит до обоих микрофонов.

### Большой residual

```
High residual alignment error: 156.2 ms
```

Дрейф может быть не идеально линейным — попробуйте стратегию 2 или 3 (следите за auto-strategy warning'ом, он назовёт лучший вариант); убедитесь, что якоря покрывают весь клип; попробуйте `large-v3` вместо маленькой модели или `--mode quality` для сложного звука.

### Кэш транскрипций

Кэш живёт в `~/.cache/whispersync/` (на других ОС — платформенный аналог). Ключ учитывает *фактическое* устройство/compute type, поэтому GPU-прогон и прогон с CPU-фолбэком никогда не перепутают кэш. Сброс: `--no-cache` на один прогон, удаление папки или `cache_max_age_days` для автопрунинга.
